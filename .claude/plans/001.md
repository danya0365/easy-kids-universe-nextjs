## ระบบเศรษฐกิจร่วม (pure rules ทั้งหมดอยู่ domain)

### ⭐ ดาว

- ต่อด่านต่อเกม best-of: ผิด 0–1 = 3⭐, 2–4 = 2⭐, 5+ = 1⭐ (`computeStars(mistakes)`)
- ปลดด่าน: `unlocked(g, n) = n === 1 || stars[g][n-1] >= 1` — ทุกด่านฟรี

### ❤️ Energy (หลอดเดียวทั้งจักรวาล — Candy Crush แบบไม่กดดัน)

- สูงสุด **10 ❤️** · หัก 1 เมื่อเริ่มด่านที่**ยังไม่เคยผ่าน** (เกมไหนก็ได้) · **เล่นซ้ำด่านที่ผ่านแล้วฟรีไม่จำกัด**
- +1 ทุก **5 นาที** (lazy regen จาก `lastRegenAt` ตอน rehydrate/interval) · 3⭐ ครั้งแรกของด่าน → +1 ❤️
- **ของขวัญรายวัน 🎁**: เปิดแอปครั้งแรกของวัน → เติมเต็มหลอด + **+2 💎** + toast
- หมด → modal ปลอบใจ + countdown — **ไม่มี**ขายเติม ไม่บล็อกการเล่นซ้ำ

### 💎 เพชร (ใหม่ — earn-only, ไม่มีเงินจริง)

- ได้จาก: ผ่านด่าน**ครั้งแรก** +5 · ได้ 3⭐ **ครั้งแรก**ของด่าน +5 (โบนัสแยก) · achievement unlock · ของขวัญรายวัน +2 — replay ไม่ให้เพชร (กัน farming)
- ใช้ซื้อใน **Diamond Shop**: ธีม + buddy (ตัวละครคู่ใจแสดงบน Home/HUD)
- track `earnedTotal`/`spentTotal` แยกจาก `balance` → รองรับ achievement "สะสมเพชร" + merge idempotent ตอนมี backend

| Cosmetic (`cosmetics.master.ts`)     | ราคา                               |
| ------------------------------------ | ---------------------------------- |
| ธีม `candy` 🍬                       | 80 💎                              |
| ธีม `galaxy` 🌌                      | 120 💎                             |
| Buddy: 🐰 กระต่าย / 🐱 แมว / 🦖 ไดโน | ตัวละ 40 💎 (🐼 แพนด้าฟรี default) |

### 🏆 Achievements (ข้ามเกม + รางวัลเพชร + toast)

| id                                             | ชื่อ (ไทย)                                | เงื่อนไข (`condition.kind`) | รางวัล       |
| ---------------------------------------------- | ----------------------------------------- | --------------------------- | ------------ |
| `first-step`                                   | ก้าวแรกสู่จักรวาล                         | `levels-completed ≥ 1`      | 10 💎        |
| `star-10` / `star-30`                          | นักสะสมดาว                                | `total-stars ≥ 10 / 30`     | 10 / 15 💎   |
| `perfect-5` / `perfect-10`                     | เพอร์เฟกต์!                               | `three-star-count ≥ 5 / 10` | 15 / 25 💎   |
| `explorer-3`                                   | นักสำรวจ — เล่นครบ 3 เกม                  | `games-played ≥ 3`          | 20 💎        |
| `abc-master` / `math-master` / `colors-master` | จบเกมนั้นครบทุกด่าน                       | `game-complete` + `gameId`  | 20 💎 ต่อเกม |
| `diamond-100`                                  | เศรษฐีเพชร                                | `diamonds-earned ≥ 100`     | 20 💎        |
| `universe-champion`                            | แชมป์จักรวาล — 3⭐ ทุกด่านทุกเกม playable | `all-perfect`               | 50 💎        |

- **Pure evaluation ใน domain**: `evaluateAchievements(snapshot, defs, unlockedIds) → AchievementDef[]` (คืนเฉพาะที่เพิ่งปลด ไม่ปลดซ้ำ) + `buildAchievementSnapshot(...)`
- Flow: จบด่าน → saveStars → grant เพชร → build snapshot → evaluate → grant รางวัล + queue **AchievementToast** (ทีละใบ ต่อคิวได้) — orchestrate ใน `presentation/lib/economy.ts` จุดเดียว

## Engines

**`use-round-engine.ts` (ใช้ร่วม 3 เกม):** `'loading'` → (useEffect เตรียมรอบ/สุ่ม — กัน hydration mismatch) → `'idle'` → ตอบ → `'checking'` → ถูก = `'correct'` (ฉลอง + chime + อ่านเสียง) → รอบถัดไป | ครบ 5 รอบ → `'complete'` · ผิด = `'wrong'` (mistakes+1, buzz+shake) → `'idle'` — engine รับ callback `checkAnswer` + expose `mistakes`, ตอน complete เรียก `economy.completeLevel(gameId, level, computeStars(mistakes))`

- **abc/spelling-round**: tray/slots เทียบ**ตัวอักษร**ไม่ใช่ tile id (รองรับ EGG/BEE), แตะช่องคืน tile, TTS อ่านตัวอักษร/คำ
- **math/math-round**: โจทย์ใหญ่ (emoji row หรือสมการ) + ปุ่มเลข 3 ตัว chunky
- **colors/colors-round**: prompt + วงกลมสีใหญ่ (hex จาก COLOR_PALETTE ผ่าน inline style — ไม่ใช่ theme token)
- AudioContext lazy ใน user gesture · speech = enhancement (iOS ต้อง gesture)
